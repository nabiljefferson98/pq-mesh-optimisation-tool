# docs/ — Project Documentation

This directory contains all narrative and analytical documentation for the PQ Mesh Optimisation project. It is organised into top-level reference documents and an experimental results subdirectory.

---

## Top-level documents

| File | Purpose |
| ---- | ------- |
| `architecture.md` | System architecture — layered module reference, data flow diagram, key design decisions, test coverage summary |
| `methodology.md` | Mathematical methodology — energy formulation (E_p, E_f, E_c, E_a), analytical gradients, L-BFGS-B algorithm, preprocessing pipeline, fabrication export, error handling |
| `future_works.md` | Candidate improvements from 2024–2026 literature — tagged as `[IMPLEMENT NOW]`, `[FUTURE WORK]`, or `[OUT OF SCOPE]`; suitable for dissertation Sections 5–6 |
| `LOGBOOK.md` | Weekly progress log — session-by-session record of implementation decisions, bugs encountered, and resolutions |

---

## results/

Structured experimental results supporting the dissertation evaluation chapter.

| File | Experiment |
| ---- | ---------- |
| `RESULTS_INDEX.md` | Master index — quick-reference summary table of all five experiments |
| `EXP-01_scalability.md` | Scalability benchmark — wall-clock time and memory vs. mesh size (3×3 to 75×75); empirical complexity O(n^1.27) |
| `EXP-02_backend_comparison.md` | Backend comparison — NumPy vs. Numba vs. CuPy speedup; numerical equivalence validation |
| `EXP-03_convergence.md` | Convergence analysis — per-iteration energy decomposition and gradient norm for 5×5, 10×10, 20×20 meshes |
| `EXP-04_weight_sensitivity.md` | Weight sensitivity — systematic sweeps over w_p, w_f, w_c; justification for default configuration |
| `EXP-05_realworld.md` | Real-world datasets — Stanford Bunny, ABC CAD, Thingi10K facade; planarity improvement statistics |

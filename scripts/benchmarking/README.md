# scripts/benchmarking — Performance Benchmarking

These scripts measure wall-clock time, peak memory usage, iteration
count, and convergence quality across a range of mesh sizes and
hardware backends. They produce the JSON output files that the
`scripts/analysis/` scripts consume to generate dissertation tables
and figures.

---

## Script Reference

| Script | Experiment | What it does |
|---|---|---|
| `benchmark_optimisation.py` | EXP-01, EXP-02 | Runs the L-BFGS-B optimiser on synthetic noisy grid meshes (Series A, generated by `scripts/mesh_generation/`) or on any OBJ mesh passed via `--mesh` (Series B). Records per-run wall-clock time, iteration count, final energy, energy reduction ratio, and the active backend. Appends results to `data/output/benchmarks/performance_data.json`. |
| `stress_test.py` | EXP-01 (supplementary) | Finds the practical upper mesh-size limit for the current machine by running on exponentially larger grids until a configurable per-mesh timeout is exceeded. Reports peak RAM usage per run and the largest mesh size that completes within the time budget. |

---

## Usage

```bash
# ── EXP-01 Series A ── synthetic scalability sweep (default mesh sizes)
python scripts/benchmarking/benchmark_optimisation.py

# ── EXP-01 Series B ── oloid resolution series
python scripts/benchmarking/benchmark_optimisation.py \
    --mesh data/input/reference_datasets/oloid/oloid64_quad.obj \
    --mesh data/input/reference_datasets/oloid/oloid256_quad.obj \
    --mesh data/input/reference_datasets/oloid/oloid1024_quad.obj \
    --mesh data/input/reference_datasets/oloid/oloid4096_quad.obj

# ── EXP-02 ── backend comparison (run once per backend)
PQ_BACKEND=numpy  python scripts/benchmarking/benchmark_optimisation.py
PQ_BACKEND=numba  python scripts/benchmarking/benchmark_optimisation.py
PQ_BACKEND=cupy   python scripts/benchmarking/benchmark_optimisation.py

# ── Stress test ── find upper mesh-size limit on the current machine
python scripts/benchmarking/stress_test.py
python scripts/benchmarking/stress_test.py --max-n 200 --timeout 60
```

### Backend Environment Variable

The `PQ_BACKEND` environment variable selects the numerical backend used
by the optimiser:

| Value | Backend | Requirement |
|---|---|---|
| `numpy` | Pure NumPy (default) | None |
| `numba` | JIT-compiled via Numba | `pip install numba` |
| `cupy` | CUDA GPU via CuPy | CUDA toolkit + `pip install cupy` |

If `PQ_BACKEND` is unset, the optimiser defaults to `numpy`.

## Output

All results are written to `data/output/benchmarks/performance_data.json`.
This file is consumed by `scripts/analysis/summarise_and_export_results.py`
and the plot scripts to produce dissertation tables and figures.

See `docs/results/EXP-01_scalability.md` and
`docs/results/EXP-02_backend_comparison.md` for the corresponding
experiment documents and expected result summaries.
